# 系统使用指南

## 📦 快速安装与启动

### macOS / Linux

1. **打开终端，进入项目目录**
   ```bash
   cd /Users/zhangxinyu/Documents/招标审计规则库/风险评估系统
   ```

2. **运行启动脚本**
   ```bash
   python3 run.py
   ```

3. **打开浏览器**
   自动打开或手动访问：`http://127.0.0.1:5000`

### Windows

1. **打开命令提示符，进入项目目录**
   ```cmd

   ```

2. **运行启动脚本**
   ```cmd
   python run.py
   ```

3. **打开浏览器**
   访问：`http://127.0.0.1:5000`

## 🎯 使用流程

### 第一步：上传文件
1. 点击**上传区域**或**选择文件**按钮
2. 选择要评估的招标文件（支持 PDF、Word、TXT）
3. 文件会自动开始分析

### 第二步：查看评估结果
系统会在 5-30 秒内显示评估结果，包括：

- **综合风险评分**（0-100）
- **风险等级**（低/中等/高/严重）
- **详细发现**（所有检测到的风险项）
- **统计信息**（各级别风险数量）

### 第三步：下载报告
支持三种报告格式：

- **HTML 报告**：美观可视化，可在浏览器查看和打印
- **TXT 报告**：纯文本格式，便于复制和编辑
- **Excel 报告**：结构化数据，便于进一步分析

## 📊 评估指标说明

### 风险等级与分数

| 风险等级 | 分数范围 | 颜色 | 含义 |
|---------|---------|-----|------|
| 低风险 | 80-100 | 绿色 | 基本符合规范 |
| 中等风险 | 60-79 | 黄色 | 存在一定缺陷 |
| 高风险 | 40-59 | 橙色 | 存在明显违规 |
| 严重风险 | 0-39 | 红色 | 存在重大违规 |

### 指标级别

系统采用四级指标分类：

| 级别 | 含义 | 扣分 | 示例 |
|------|------|------|------|
| **A级** | 重大违规 | -10分 | 虚假招标人、重复投标 |
| **B级** | 严重违规 | -5分 | 招标文件矛盾、评标不规范 |
| **C级** | 中等风险 | -2分 | 信息公示不足 |
| **D级** | 低风险 | -1分 | 流程改进建议 |

## 🔍 详细发现说明

每条风险发现包含以下信息：

### 发现内容
- **规则ID**：如 M1.1（模块.序号）
- **级别**：A/B/C/D
- **风险名称**：简要描述
- **风险描述**：详细说明
- **匹配关键词**：文件中出现的触发词
- **置信度**：匹配程度（百分比）
- **整改建议**：具体改进方向

### 示例
```
❌ M1.1 [A级] 招标人身份和资格不符
   描述: 招标人不具有相应的资质或法人地位，违反招标人条件
   匹配: 虚假招标人, 假冒身份, 身份造假
   置信: 85%
   💡 整改建议: 立即停止招标，核实招标人身份和资格...
```

## 📈 7 个评估模块

### 1️⃣ 模块一：招标人自主权风险（11项）
- 招标人资格审查
- 招标预案制定
- 采购需求论证
- 招标方式选择
- 资金合法性
- 等等

**高风险指标**：虚假身份、资金来源不合法、虚假承诺

### 2️⃣ 模块二：市场准入与投标资格风险（7项）
- 投标资格审查
- 投标保证金管理
- 重复投标检测
- 联合体管理
- 黑名单查询

**高风险指标**：黑名单企业参标、重复投标、资格造假

### 3️⃣ 模块三：标准招标文件风险（6项）
- 文件内容完整性
- 技术规范合理性
- 评标标准清晰性
- 商务条件公平性
- 合同条款平衡性

**高风险指标**：内容矛盾、标准不清、倾向性条款

### 4️⃣ 模块四：定标权保护风险（5项）
- 评标委员会组成
- 专家选择规范
- 评标程序规范
- 定标权行使
- 结果公告及时性

**高风险指标**：评委有利益关系、评标程序不当

### 5️⃣ 模块五：信用评价应用风险（5项）
- 黑名单查询
- 失信企业管理
- 信用评价标准
- 信用信息使用
- 修复机制完善

**高风险指标**：黑名单企业参标、信用评价不公

### 6️⃣ 模块六：交易监管与服务风险（5项）
- 交易场所规范
- 代理机构选择
- 监管部门监督
- 交易服务收费
- 电子系统使用

**高风险指标**：场所不规范、监管缺失

### 7️⃣ 模块七：保证金相关风险（6项）
- 招标保证金管理
- 投标保证金管理
- 保证金退回及时性
- 履约保证金规范
- 保证金支付程序
- 保证金账户管理

**高风险指标**：保证金超标、退回不及时

## 🛠️ 常见问题与解决

### Q1：系统启动不了怎么办？

**A：** 按照以下步骤排查：

```bash
# 1. 检查 Python 版本
python --version  # 需要 3.8+

# 2. 手动安装依赖
pip install -r requirements.txt

# 3. 手动启动
python app.py
```

### Q2：文件上传失败怎么办？

**A：** 检查以下问题：
- 文件大小是否超过 50MB
- 文件格式是否正确（PDF/DOCX/DOC/TXT）
- 文件是否损坏或无法读取

### Q3：评估结果不准确怎么办？

**A：** 
- 系统使用关键词匹配，精准度依赖于文件内容
- 建议参考完整规则库进行人工复核
- 对低置信度的发现需要特别注意

### Q4：可以定制规则吗？

**A：** 可以。编辑 `rules_engine.py` 文件，修改 `_initialize_rules()` 方法中的规则定义。

### Q5：报告可以定制样式吗？

**A：** 可以。编辑 `report_generator.py` 文件中的 HTML/CSS 部分。
