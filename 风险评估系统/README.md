# 招标投标风险评估系统

一个智能化的招标投标风险自动评估系统，支持 PDF、Word、TXT 等多种文件格式，提供全面的 45 项风险指标检查。

## 🎯 系统特点

- **全面评估**：涵盖 45 项风险指标，7 个评估模块
- **多格式支持**：支持 PDF、DOCX、DOC、TXT 文件
- **自动分析**：关键词匹配 + 置信度评分
- **详细报告**：HTML、TXT、Excel 三种报告格式
- **快速反馈**：秒级评估结果
- **明确建议**：每项风险都有具体整改建议

## 📋 评估模块

### 模块 1: 招标人自主权风险 (M1.1-M1.11)
- 招标人身份、资格验证
- 招标预案制定和执行
- 采购需求论证
- 资金来源和使用规范

### 模块 2: 市场准入与投标资格风险 (M2.1-M2.7)
- 投标资格审查标准
- 投标保证金规范
- 重复投标风险
- 联合体投标规范

### 模块 3: 标准招标文件风险 (M3.1-M3.6)
- 招标文件内容完整性
- 评标标准合规性
- 技术规范合理性
- 价格条款规范性

### 模块 4: 定标权保护风险 (M4.1-M4.5)
- 评标委员会组成
- 评标程序规范
- 定标权行使
- 评标专家选择

### 模块 5: 信用评价应用风险 (M5.1-M5.5)
- 信用记录查询
- 黑名单管理
- 不良记录使用
- 信用等级应用

### 模块 6: 交易监管与服务风险 (M6.1-M6.5)
- 交易场所规范
- 监管部门监督
- 交易服务收费
- 电子招标系统使用

### 模块 7: 保证金相关风险 (M7.1-M7.6)
- 招标保证金规范
- 投标保证金管理
- 履约保证金使用
- 保证金退回规范

## 🚀 快速开始

### 环境要求

- Python 3.8+
- pip 包管理器

### 安装步骤

1. **克隆或下载项目**
   ```bash
   cd 风险评估系统
   ```

2. **创建虚拟环境**（推荐）
   ```bash
   python -m venv venv
   source venv/bin/activate  # macOS/Linux
   # 或
   venv\Scripts\activate  # Windows
   ```

3. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

4. **启动应用**
   ```bash
   python app.py
   ```

5. **打开浏览器**
   访问 `http://127.0.0.1:5000`

## 📖 使用指南

### 基本流程

1. **上传文件**
   - 点击上传区域或拖拽文件
   - 支持 PDF、Word、TXT 格式
   - 最大文件大小 50MB

2. **等待评估**
   - 系统自动进行风险分析
   - 通常需要 5-30 秒（取决于文件大小）

3. **查看结果**
   - 综合风险评分（0-100）
   - 风险等级判断
   - 详细发现列表
   - 具体整改建议

4. **下载报告**
   - HTML 报告：Web 查看
   - TXT 报告：纯文本
   - Excel 报告：数据分析

### 风险等级说明

| 等级 | 分数范围 | 含义 | 建议 |
|------|--------|------|------|
| 低风险 | 80+ | 基本符合规范 | 继续监控 |
| 中等风险 | 60-79 | 存在一定风险 | 改进流程 |
| 高风险 | 40-59 | 存在明显风险 | 立即整改 |
| 严重风险 | <40 | 存在重大违规 | 紧急处理 |

### 评估指标级别

- **A 级**：重大违规，必须立即整改，扣 10 分
- **B 级**：严重违规，需要限期改正，扣 5 分
- **C 级**：中等风险，应该改进，扣 2 分
- **D 级**：低风险或建议项，扣 1 分

## 🔧 系统架构

```
风险评估系统/
├── app.py              # Flask 主应用
├── rules_engine.py     # 风险规则引擎
├── file_processor.py   # 文件处理模块
├── report_generator.py # 报告生成模块
├── requirements.txt    # 依赖清单
├── templates/
│   └── index.html      # Web 界面
├── uploads/            # 文件上传目录
└── reports/            # 报告输出目录
```

### 核心模块说明

#### rules_engine.py
- 包含 45 项风险指标定义
- 每项指标包含 10-15 个检测关键词
- 提供评分和统计分析功能

#### file_processor.py
- 支持多种文件格式
- 自动编码识别
- 文本提取和清理

#### report_generator.py
- 生成多种格式报告
- 包含详细的风险描述
- 提供整改建议

#### app.py
- Flask 应用框架
- RESTful API 接口
- 文件上传处理

## 📝 API 文档

### 上传评估
```
POST /api/assess
Content-Type: multipart/form-data

参数：
- file: 待评估的文件

返回：
{
    "success": true,
    "data": {
        "filename": "招标文件.pdf",
        "risk_score": 65,
        "risk_level": "中等风险",
        "stats": {...},
        "findings": [...]
    }
}
```

### 获取规则
```
GET /api/rules

返回：45 项风险指标的详细定义
```

### 获取系统统计
```
GET /api/statistics

返回：系统规则的统计信息
```

## 🛠️ 常见问题

### Q1: 能否离线使用？
**A:** 可以。应用在本地运行，不需要网络连接。

### Q2: 如何扩展规则？
**A:** 编辑 `rules_engine.py` 中的 `_initialize_rules()` 方法，添加新的指标。

### Q3: 支持哪些文件格式？
**A:** 目前支持 PDF、DOCX、DOC、TXT，可根据需要扩展。

### Q4: 评估需要多长时间？
**A:** 通常 5-30 秒，取决于文件大小和内容复杂度。

### Q5: 可以对报告进行定制吗？
**A:** 可以。编辑 `report_generator.py` 中的模板和样式。
